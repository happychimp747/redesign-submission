Redesign: Usability Focused Redesign for a Website

Context

For this assignment in my UIUX class, I was tasked to choose a website with usability issues and make a new redesign.

I decided to choose a government website. After browsing through numerous federal and local government websites, I discovered that most of them are well-designed. However, the judiciary branch’s website still suffers from occasional usability issues; specifically, I chose to analyze and remake the Rhode Island bar admissions website.

View the Original Website ↓

A Summary of Usability Issues

  1. The top usability concern: Buttons do not follow a clear conceptual model.
    • Currently, there are lists of buttons on the left and right. It is not immediately clear which list users should use for navigation.
    • It will take time for users to figure out that some buttons are quick links to important resources.
    • It’s not clear that most of the buttons on the left are detailed information on the 5 common admission types.
  1. It does not offer a button for the bar admissions homepage.
      Users should feel safe to leave the bar exam homepage and then reenter.
  2. Main text is slightly gray. It makes reading a little bit more inefficient.
      The website is intended for applicants to read in detail. Every measure should be taken to make sure the applicants read as best as they could.
  3. The list of types of admissions is not readable.
      There should be a header for each list item.
  4. The contact information section distracts from the main text.
  5. COVID information section offers low-value information.
      Its only purpose should be to tell readers that there are changes to rules based on COVID-19. Users should read specific rules at their own liberty.

Accessibility Audit

To identify accesibility issues on the original website, I used WebAim Wave. I agree with its findings that the headers have low-contrast. It also found that most of the buttons, implemented with unordered lists, have accessibility issues.

Many links on the website are to pdfs and the accessibility evaluation tool gives warnings on them. Because it is conventional for law-related texts to be in pdfs, I disagree that they are problematic for this website.

Low-definition Mockup

Using Balsamiq Wireframes, I produced the following mockup. The numberings correspond to the usability concerns I described earlier.

  1. Navigation elements are placed on the right; all information links are in the bottom.

  2. A general information button assures users with a way back as they navigate to other sub-sections.

  3. Use high contrast text.

  4. Use headers and nested lists to achieve hierarchy.

  5. Contacts is placed at the page footer.

  6. Hide most of COVID information but still alert users about it.

Lessons Learned

Heavy information density is a usability challenge specific to government agency websites. If "quick links" are necessary, it is critical to make sure that they are visually distinct from other navigational elements.

Bootstrap provides a sound method to implement websites responsive to all screen sizes, thanks to its meticulous rules to adjust HTML element widths.

Modifying Bootstrap based css rules could be complex. A quick solution is to use Chrome Develepor to find the relevant Bootstrap CSS rules and override them to the website's design. In addition, use !important tags to make sure that these overrides are successful.
